Introdução ao Docker: Containerização para Desenvolvedores

O que é Docker?

Docker é uma plataforma que facilita a criação, distribuição e execução de aplicações dentro de containers — ambientes isolados que garantem que o software funcione de maneira consistente, independente do sistema operacional host.

Por que usar Docker?

  • Ambientes padronizados e portáteis.
  • Evita o clássico problema de "na minha máquina funciona".
  • Facilita a implantação e escalabilidade de aplicações.
  • Permite gerenciar dependências sem conflito.

Conceitos básicos

  • Imagem (Image): Template imutável com sistema operacional, bibliotecas e seu aplicativo.
  • Container: Instância em execução da imagem, isolada e leve.
  • Dockerfile: Arquivo de texto que define como construir uma imagem.
  • Docker Hub: Repositório público para compartilhar imagens.

Instalando o Docker

Você pode instalar o Docker no Windows, macOS e Linux diretamente do site oficial:

https://www.docker.com/get-started

Comandos básicos do Docker

docker --version
docker run hello-world
docker ps
docker images
docker stop [container_id]
docker rm [container_id]
docker rmi [image_id]

Exemplo rápido: rodando um servidor Nginx

docker run -d -p 8080:80 --name meu-nginx nginx

Esse comando faz o Docker baixar a imagem oficial do Nginx, criar e executar um container em background, e expor a porta 80 do container na porta 8080 do seu computador.

Agora, acesse http://localhost:8080 no seu navegador para ver a página padrão do Nginx.

Criando sua própria imagem com Dockerfile

Exemplo de um Dockerfile simples para uma aplicação Node.js:

FROM node:16-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]

Para construir a imagem, use:

docker build -t minha-app-node .

E para executar o container:

docker run -p 3000:3000 minha-app-node

Boas práticas

  • Use imagens oficiais e pequenas para reduzir tamanho e vulnerabilidades.
  • Evite rodar processos como root dentro do container.
  • Documente seu Dockerfile e organize suas imagens com tags.
  • Limpe imagens e containers antigos regularmente para economizar espaço.
