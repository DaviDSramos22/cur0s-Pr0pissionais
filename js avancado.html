JavaScript Avançado: Promises, Async/Await e APIs

Início Tutoriais Sobre Contato

Este tutorial explora conceitos avançados de JavaScript, especialmente programação assíncrona com Promises e Async/Await, além de como consumir APIs REST para construir aplicações dinâmicas e robustas.

Promises

Uma Promise é um objeto que representa a eventual conclusão (ou falha) de uma operação assíncrona e seu valor resultante.

const minhaPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Sucesso!');
  }, 2000);
});

minhaPromise.then(resultado => {
  console.log(resultado); // Sucesso!
}).catch(erro => {
  console.error(erro);
});

Async/Await

Async/Await é uma sintaxe que simplifica o uso de Promises, tornando o código assíncrono mais legível e parecido com código síncrono.

async function fetchDados() {
  try {
    const response = await fetch('https://api.exemplo.com/dados');
    const dados = await response.json();
    console.log(dados);
  } catch (error) {
    console.error('Erro:', error);
  }
}

fetchDados();

Consumindo APIs REST

APIs REST permitem a comunicação entre cliente e servidor usando requisições HTTP. Com fetch, podemos obter dados facilmente.

fetch('https://api.exemplo.com/usuarios')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

Tratamento de erros

É fundamental capturar erros para garantir que a aplicação não quebre e possa responder apropriadamente.

try {
  await algumaOperacaoAssincrona();
} catch (error) {
  console.error('Ocorreu um erro:', error);
}
